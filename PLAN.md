# Modular Architecture Plan

## Goal
Break the monolithic `index.html` (1808 lines) into separate component files without a build step. Use native ES modules and inline `<script type="module">` for clean imports.

## Architecture

```
src/
├── index.html              # Entry point (imports all components)
├── components/
│   ├── Header.js           # Top bar: logo, search, nav buttons, resources dropdown
│   ├── SlidePanel.js       # Slide carousel: step badge, title, body, navigation
│   ├── Sidebar.js          # Side panel: workspace tree, quick commands, repo links
│   ├── ProgressBanner.js   # Completion progress bar
│   ├── NavigationMap.js    # Steps grid overview
│   ├── VideoSection.js     # YouTube tutorial embed
│   ├── MarkdownViewer.js   # Modal for rendering info/*.md files
│   ├── Toast.js            # Notification toast system
│   └── MobileDrawer.js     # Mobile side panel drawer + backdrop
├── utils/
│   ├── storage.js          # localStorage helpers (progress, settings)
│   ├── search.js           # Global search across slides/commands
│   └── helpers.js          # copyCommand, escapeHtml, showToast
└── data/
    └── slides.js           # slides[] array extracted from inline JS
```

## Migration Steps

### Phase 1: Extract data & utilities (no visual change)
1. Create `data/slides.js` — export the `slides[]` array
2. Create `utils/storage.js` — localStorage wrapper for completed slides
3. Create `utils/search.js` — search logic across slides/commands
4. Create `utils/helpers.js` — copyCommand, escapeHtml, showToast, etc.

### Phase 2: Extract components (visual change, one at a time)
5. Create `components/Header.js` — header DOM + event handlers
6. Create `components/Toast.js` — toast notification
7. Create `components/MarkdownViewer.js` — modal + renderMarkdown
8. Create `components/SlidePanel.js` — slide rendering + navigation
9. Create `components/Sidebar.js` — workspace tree, quick commands, repo links
10. Create `components/ProgressBanner.js` — completion progress
11. Create `components/NavigationMap.js` — steps grid
12. Create `components/VideoSection.js` — video embed
13. Create `components/MobileDrawer.js` — mobile drawer + backdrop

### Phase 3: Wire up
14. Update `index.html` to `<script type="module">` approach
15. Each component imports its dependencies and exports an init function
16. `index.html` calls init functions in order

## Key Design Decisions

### No build step
- Pure ES modules via `type="module"` — works in all modern browsers
- No Webpack, Vite, or bundler needed
- `import`/`export` syntax only

### Component pattern
```js
// components/Toast.js
export function initToast(container) {
  // Create DOM, attach event listeners
  // Return public API (show, hide)
}
```

### State management
- Use a shared state object passed between components (or a simple event bus)
- Alternatively, keep global state on `window` for backward compat with inline `onclick`

### CSS
- Keep Tailwind classes in HTML (generated by component JS)
- Keep custom CSS in `<style>` in index.html or a shared `styles/globals.css`

## Risks
- `onclick` handlers in HTML must reference globals — need to attach to `window` in module scope
- ES modules require `type="module"` which changes script loading behavior (deferred by default)
- Some components (Sidebar, SlidePanel) are tightly coupled — extract carefully
